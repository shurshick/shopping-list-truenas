# API сервера

Актуально для версии `v1.3.9`.

Базовый адрес: `https://your-domain.example`

## Авторизация

- `POST /auth/register` - зарегистрировать пользователя.
- `POST /auth/login` - войти в аккаунт.

Обычная регистрация пользователей доступна только после завершения мастера `/setup`.

Для `POST /auth/login` и `POST /auth/register` действует базовое in-memory ограничение частоты запросов по IP и email. При превышении лимита сервер возвращает `429` с сообщением на русском языке.

Тело запроса:

```json
{
  "email": "user@example.com",
  "password": "secret123"
}
```

Ответ:

```json
{
  "access_token": "...",
  "token_type": "bearer"
}
```

## Синхронизация

`GET /sync`

Заголовок:

```text
Authorization: Bearer <token>
```

## Идемпотентность клиентских операций

Новые Android-клиенты отправляют для операций offline-очереди UUID `client_operation_id`. Его можно передать в теле `POST /lists` и `POST /lists/{list_id}/items`, а также в заголовке:

```text
X-Client-Operation-Id: <uuid>
```

Если сервер уже применил операцию с таким `client_operation_id` для текущего пользователя, повторный запрос возвращает сохраненный результат и не выполняет действие второй раз. Это защищает от дублей, когда сервер успел создать список или товар, но клиент не получил ответ из-за сетевого сбоя.

Для `create_list` и `create_item` можно передавать `temp_id` из Android. Сервер сохраняет соответствие `temp_id` и созданного серверного ID в таблице `client_operations`.

Старые клиенты без `client_operation_id` работают как раньше: операция выполняется обычным способом и не сохраняется в таблицу replay.

## Публичная конфигурация сервера

`GET /server-config`

Ответ:

```json
{
  "app_name": "Список покупок",
  "external_url": "https://shopping.example.com",
  "allow_registration": true,
  "setup_completed": true
}
```

Ответ `/server-config` не содержит секретов, паролей, JWT-ключей и параметров подключения к базе данных.

## Защита веб-форм

HTML-форма `/setup` использует CSRF-токен и проверку `Origin`/`Referer` для POST-запроса. Эта защита относится к браузерным HTML-действиям и не применяется к мобильному API с Bearer-токеном.

Rate limit сервера хранится в памяти процесса и сбрасывается при перезапуске контейнера.

## Наблюдаемость

- `GET /health` - состояние API, версия сервера, доступность БД, текущая миграция и время сервера.
- `GET /admin` - веб-страница администратора для входа и просмотра состояния сервера в браузере.
- `GET /admin/status` - административная статистика сервера. Требуется токен администратора.
- `DELETE /admin/activity` - очистить всю историю действий. Требуется токен администратора.

## Списки

- `POST /lists` - создать список.
- `PATCH /lists/{list_id}` - переименовать список, доступно владельцу.
- `DELETE /lists/{list_id}` - удалить список. Для владельца удаляет список полностью, для участника убирает только его доступ.
- `POST /lists/{list_id}/copy` - создать копию списка для текущего пользователя.
- `DELETE /lists/{list_id}/items` - очистить список от всех позиций.
- `DELETE /lists/{list_id}/items/checked` - удалить только позиции, отмеченные как купленные.
- `PATCH /lists/{list_id}/items/checked` - снять отметку покупки со всех купленных позиций.
- `GET /lists/{list_id}/members` - посмотреть участников списка.
- `GET /lists/{list_id}/activity` - посмотреть последние события истории списка.
- `DELETE /lists/{list_id}/activity` - очистить историю действий выбранного списка.
- `POST /lists/{list_id}/share` - открыть доступ другому зарегистрированному пользователю.
- `POST /lists/{list_id}/invite` - создать одноразовую ссылку-приглашение со сроком действия.
- `POST /invites/{token}/accept` - принять одноразовое приглашение до окончания срока действия.
- `GET /join/{token}` - страница приглашения, которая открывает Android-приложение.

Срок действия приглашений задаётся переменной окружения `INVITE_TOKEN_HOURS`. Значение по умолчанию: `168` часов.

## Android-клиент

- Минимальная версия Android: Android 8.0 Oreo.
- Минимальный API: 26.
- Целевая версия сборки: Android 15, API 35.

## Товары

- `POST /lists/{list_id}/items` - добавить покупку.
- `PATCH /items/{item_id}` - изменить название, количество или отметку покупки.
- `DELETE /items/{item_id}` - удалить покупку.
