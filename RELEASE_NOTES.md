# Список покупок v1.3.7-security-hardening

Актуальный публичный релиз Android-приложения и серверной части для самостоятельного размещения.

## Изменения в v1.3.7-security-hardening

- Android-приложение хранит токен входа в защищенном `EncryptedSharedPreferences`.
- При обновлении со старой версии токен автоматически переносится из обычных настроек в защищенное хранилище, старое значение удаляется.
- Пароль после входа или регистрации не сохраняется в состоянии приложения.
- Выход из аккаунта удаляет токен и чувствительные данные.
- OkHttp logging interceptor подключается только в debug-сборках, в release-сборке сетевые логи не включаются.
- В debug-логах скрывается заголовок `Authorization`.
- На сервере добавлено базовое in-memory ограничение частоты запросов для `POST /auth/login`, `POST /auth/register` и `POST /setup`.
- HTML-форма `/setup` защищена CSRF-токеном и проверкой `Origin`/`Referer`.
- Принятие приглашения стало безопаснее при одновременных запросах: приглашение выбирается с блокировкой строки, повторное принятие отклоняется.
- Добавлены backend-тесты для health, setup, auth, rate limit, прав доступа, одноразовых приглашений, публичной конфигурации без секретов и CSRF-защиты.
- Добавлен GitHub Actions workflow `Backend CI`: pytest-тесты backend и Docker-сборка серверной части.
- Android release workflow теперь собирает release APK, а не debug APK.
- Обновлены номера версии Android-приложения и API до `1.3.7`.

## Важное про безопасность

- Rate limit хранится в памяти процесса и сбрасывается после перезапуска контейнера.
- Для доступа из интернета используйте HTTPS через reverse proxy.
- Мобильный API с Bearer-токеном не требует CSRF-токена.
- Подробности описаны в `SECURITY.md`.

## Что входит

- Android-приложение для совместных списков покупок.
- Сервер синхронизации на FastAPI и PostgreSQL.
- Веб-мастер первичной настройки сервера по адресу `/setup`.
- Веб-страница администратора по адресу `/admin`.
- Обязательное создание администратора при первом запуске.
- Автоматические миграции базы данных при запуске контейнера API.
- Несколько способов запуска сервера: Docker Compose, сборка напрямую из GitHub, готовый GHCR-образ и TrueNAS Custom App.
- APK подписывается постоянным ключом проекта, поэтому публичные версии устанавливаются поверх друг друга.

## Возможности приложения

- Регистрация и вход пользователей.
- Несколько списков покупок.
- Совместный доступ по email.
- Одноразовые ссылки-приглашения со сроком действия.
- Просмотр участников списка.
- История действий списка и очистка истории.
- Переименование, копирование, очистка и удаление списков.
- Очистка только купленных товаров.
- Возврат купленных товаров в раздел "Купить".
- Добавление, редактирование, отметка и удаление товаров.
- Отмена удаления товара с таймером 5 секунд.
- Офлайн-очередь изменений с понятным статусом синхронизации.
- Разделы "Купить" и "Куплено" на основном экране.
- Светлая, темная и системная тема оформления.
- Справочник товаров, автопоиск при вводе и автоматическое пополнение справочника при ручном добавлении.
- Экран настроек приложения.
- Окно "О приложении" с версией, ссылкой на проект и проверкой обновлений.

## Совместимость Android

- Минимальная версия Android: Android 8.0 Oreo.
- Минимальный API: 26.
- Целевая версия сборки: Android 15, API 35.

## Сервер

Сервер можно запускать на обычном Docker-сервере, VPS, домашнем Linux-сервере, NAS с Docker или через TrueNAS Custom App.

Готовый Docker-образ:

```text
ghcr.io/shurshick/shopping-list-api:latest
ghcr.io/shurshick/shopping-list-api:v1.3.7-security-hardening
```

## Проверки

```bash
python -m pytest backend/tests
cd android
./gradlew assembleDebug --stacktrace
./gradlew assembleRelease --stacktrace
```
