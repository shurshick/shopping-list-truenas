# Безопасность

## Что защищено в текущей версии

- Android-приложение хранит токен входа в `EncryptedSharedPreferences`.
- При обновлении со старой версии токен автоматически переносится из обычных настроек в защищенное хранилище, старое значение удаляется.
- Пароль не сохраняется после входа или регистрации.
- Выход из аккаунта удаляет токен и чувствительные данные.
- HTTP-логи OkHttp включаются только в debug-сборках. В release-сборке сетевое логирование не подключается.
- Заголовок `Authorization` скрывается в debug-логах.
- На сервере включено базовое in-memory ограничение частоты запросов для `/auth/login`, `/auth/register` и `/setup`.
- Веб-форма `/setup` защищена CSRF-токеном и проверкой `Origin`/`Referer`.
- Мобильный API с Bearer-токеном не требует CSRF-токена.
- Принятие приглашения выполняется с блокировкой строки приглашения, повторное использование ссылки отклоняется.

## Важные ограничения

- Rate limit хранится в памяти процесса и сбрасывается после перезапуска контейнера.
- Для доступа из интернета используйте HTTPS через reverse proxy.
- Не публикуйте сервер наружу по обычному HTTP, потому что при входе передаются email, пароль и токен авторизации.
- Для более жесткой защиты публичного сервера в будущем можно добавить Redis-rate-limit, аудит с IP-адресами и отдельные политики блокировки аккаунтов.

## Проверки перед релизом

```bash
python -m pytest backend/tests
cd android
./gradlew assembleDebug --stacktrace
./gradlew assembleRelease --stacktrace
```

Отсутствие локальной Java/JDK на компьютере разработчика не считается ошибкой исходного кода. В GitHub Actions для Android-сборки устанавливается JDK 17.
